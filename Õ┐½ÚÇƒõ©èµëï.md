# 快速上手 · 這幾個檔案怎麼用

先看懂每個檔案是什麼,再照你的需求選一條路走。

---

## 每個檔案是什麼

| 檔案 | 是什麼 | 你要不要碰它 |
|---|---|---|
| `index.html` | **App 本體**。用瀏覽器打開就能看特價、比價、追蹤。 | ✅ 這是你每天用的 |
| `data/deals.json` | **特價資料**。App 讀這個檔顯示內容。 | ⚙️ 自動更新,通常不用手動改 |
| `baby_manual.json` | **藥妝嬰兒特價**(Kruidvat/Etos)。每次更新都會併進去。 | ✏️ 想改嬰兒品項時編輯它 |
| `scrape_foldoo.py` | **爬蟲程式**。去 Foldoo 抓最新特價、寫進 `data/deals.json`。 | 🤖 給 GitHub 自動跑,你不用手動執行 |
| `requirements.txt` | 爬蟲需要的套件清單。 | 🤖 GitHub 自動安裝 |
| `.github/workflows/update.yml` | **排程表**。叫 GitHub 每週一自動跑爬蟲。 | 🤖 放上去就會自己運作 |
| `README.md` | 完整技術說明。 | 📖 需要細節時再看 |

---

## 路線 A:我只想現在就用 App(最簡單,3 步)

不碰 GitHub,資料先用現成的 59 筆。

1. 把 `index.html` 下載到手機或電腦。
2. 用瀏覽器打開它(電腦:雙擊;手機:用檔案管理員點開,或先傳到雲端硬碟再開)。
3. 就能用了 —— 看特價、按分類、加追蹤清單、新增自己的特價。

> 想放到手機主畫面像 App 一樣:在手機瀏覽器打開後,選「加入主畫面」。
> 缺點:這樣資料不會自動更新,得靠你手動新增,或用路線 B。

---

## 路線 B:我要每週自動更新(一次設定,之後全自動)

需要一個免費的 GitHub 帳號。整個過程約 10 分鐘,只做這一次。

### 第 1 步:把資料夾放上 GitHub
1. 到 github.com 註冊/登入,按右上「+」→ **New repository**。
2. 取名 `bonus-radar`,選 **Public**,按 **Create repository**。
3. 在新頁面點 **uploading an existing file**,把這整個資料夾裡的所有檔案拖進去上傳,
   按 **Commit changes**。
   > 要保持資料夾結構:`data/deals.json` 要在 data 子資料夾裡、
   > `.github/workflows/update.yml` 要在對應資料夾裡。用網頁拖拉整個資料夾即可保留。

### 第 2 步:開啟自動寫入權限
1. 你的 repo → **Settings**(上方齒輪)→ 左側 **Actions** → **General**。
2. 捲到最下 **Workflow permissions**,選 **Read and write permissions** → **Save**。
   (這樣機器人才能把更新後的資料存回來。)

### 第 3 步:先手動跑一次,確認會動
1. repo 上方 **Actions** 分頁 → 左側點「每週更新特價」→ 右側 **Run workflow** → 綠色 **Run workflow**。
2. 等一下下,點進那次執行看記錄。應該會看到每家抓到幾筆、以及「已寫入 data/deals.json」。
   > 之後每週一早上會自己跑,你什麼都不用做。

### 第 4 步:讓 App 讀雲端資料
1. 在 repo 打開 `data/deals.json`,點右上 **Raw**,複製網址列的網址,長這樣:
   ```
   https://raw.githubusercontent.com/你的帳號/bonus-radar/main/data/deals.json
   ```
2. 打開 `index.html`(電腦或手機皆可)。
3. 捲到最底,按 **設定自動來源** → 貼上剛剛的網址 → 確定。

完成!以後每次打開 App 都會自動同步最新資料,也可隨時按 **立即同步雲端**。

## 常見情況怎麼辦

**想改藥妝嬰兒的特價?**
編輯 `baby_manual.json`,照裡面的格式改(沒有固定價就填 `null`),存檔。
若用路線 B,改完在 GitHub 重新上傳這個檔即可,下次更新會帶上。

**想自己加一筆特價?**
直接在 App 裡按「新增特價」。你手動加的項目**不會**被雲端同步蓋掉。

**換手機/清了瀏覽器,資料不見?**
App 資料存在瀏覽器本機。搬家前按底部 **匯出備份 (JSON)**,到新裝置按 **匯入**。
(有設路線 B 的雲端來源就更省事,按「立即同步雲端」即可拉回。)

**每週一想手動叫我更新一輪?**
路線 A/B 都行:跟我說「幫我更新這週特價」,我查完給你一份可匯入的 JSON,
或直接更新你 repo 裡的 `data/deals.json`。
