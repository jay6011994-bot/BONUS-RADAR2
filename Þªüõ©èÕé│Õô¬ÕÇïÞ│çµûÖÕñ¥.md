# 要上傳的就是這個資料夾:bonus-radar

「傳資料夾」= 把下面這個 **bonus-radar** 資料夾裡的**全部東西**傳上 GitHub。
就是你下載我給的檔案時,最外層那個資料夾。

---

## 它長這樣(共 9 個檔案,分佈在 3 層)

```
bonus-radar/                         ← 這個資料夾(整包要上傳的就是它)
│
├── index.html                       ← App 本體(你平常打開來用的)
├── scrape_foldoo.py                 ← 爬蟲程式
├── requirements.txt                 ← 爬蟲要裝的套件清單
├── baby_manual.json                 ← 藥妝嬰兒特價(可自己編輯)
├── README.md                        ← 完整說明
├── 快速上手.md                       ← 白話上手指南
├── GitHub設定詳細教學.md              ← GitHub 一步步教學
│
├── data/                            ← 子資料夾①
│   └── deals.json                   ← 特價資料(App 讀這個)
│
└── .github/                         ← 子資料夾②(以點開頭,有些地方不顯眼)
    └── workflows/
        └── update.yml               ← 每週自動更新的排程表
```

---

## 三件要記住的事

1. **要上傳的是「bonus-radar 資料夾裡的內容」**,不是它的上一層。
   打開 bonus-radar,看到上面那 7 個檔案 + `data`、`.github` 兩個子資料夾,就是對的。

2. **`data` 和 `.github` 這兩個子資料夾一定要一起上傳**,而且要保留它們的層級。
   - 上傳後在 GitHub 要能看到 `data/deals.json`(在 data 資料夾裡)
   - 以及 `.github/workflows/update.yml`(在 .github/workflows 裡)

3. **`.github` 以點(.)開頭**,在電腦或 GitHub 首頁有時看起來不明顯,但它必須在。
   確認方法:上傳完到 GitHub 的 **Actions** 分頁,若出現「每週更新特價」,就代表它有傳上去。

---

## 怎麼確認「我全選對了」

上傳前,在電腦打開 bonus-radar 資料夾,你應該數到:
- **7 個檔案**:index.html、scrape_foldoo.py、requirements.txt、baby_manual.json、
  README.md、快速上手.md、GitHub設定詳細教學.md
- **2 個資料夾**:data、.github

全部一起選、一起拖上去,就對了。

> 看不到 `.github`?在 Windows 檔案總管:上方「檢視」→ 勾「隱藏的項目」。
> Mac 訪達:按 `Cmd + Shift + .`(句點)顯示隱藏檔。
> 或者乾脆用《GitHub設定詳細教學.md》第 2 步的「方法二」,手動打路徑 `.github/workflows/update.yml` 建立它。
